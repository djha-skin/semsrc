# Semsrc Ontology Design

## Ontology Strategy

Semsrc follows a layered ontology approach, building on established standards before adding domain-specific extensions.

### Layer 1: Foundation Standards
Reuse existing, well-understood ontologies for maximum interoperability and AI familiarity.

### Layer 2: Core Extension
Add minimal Semsrc-specific concepts that don't exist in standards.

### Layer 3: Domain Extensions
Optional, project-specific extensions for specialized use cases.

## Foundation Ontologies

### PROV-O (Provenance Ontology)
**Purpose**: Track origins and relationships between entities
**Namespace**: `http://www.w3.org/ns/prov#`
**Key Classes**:
- `prov:Entity` - Anything we want to track provenance of
- `prov:Activity` - Actions that generate or use entities
- `prov:Agent` - People, organizations, software acting
- `prov:wasGeneratedBy` - Entity generated by activity
- `prov:used` - Activity used entity
- `prov:wasAttributedTo` - Entity attributed to agent
- `prov:wasDerivedFrom` - Entity derived from another

**Semsrc Usage**:
- Commits are `prov:Activity`
- Files/blobs are `prov:Entity`
- Developers are `prov:Agent`
- Builds are `prov:Activity`
- Tickets are `prov:Entity`

### NATMUK Filesystem Ontology
**Purpose**: Represent filesystem structures and operations
**Namespace**: `http://purl.org/natmuk/fs#`
**Key Classes**:
- `fs:File` - A file with content
- `fs:Directory` - A directory containing files/directories
- `fs:Path` - Path location in filesystem
- `fs:hasContent` - File has content (blob)
- `fs:contains` - Directory contains item
- `fs:hasPath` - Item has filesystem path

**Semsrc Usage**:
- Represent Git tree structures
- Track file moving/renaming
- Model working directory state

### Dublin Core
**Purpose**: Basic metadata vocabulary
**Namespace**: `http://purl.org/dc/terms/`
**Key Properties**:
- `dcterms:created` - Creation date
- `dcterms:creator` - Creator agent
- `dcterms:description` - Description
- `dcterms:identifier` - Unique identifier
- `dcterms:title` - Title/name

### Core Classes

#### semsrc:Repository
Represents a complete Semsrc repository instance.

```turtle
semsrc:Repository a owl:Class ;
    rdfs:subClassOf prov:Entity ;
    rdfs:label "Semsrc Repository" ;
    rdfs:comment "A complete Semsrc repository containing objects and triples." ;
    rdfs:isDefinedBy <http://semsrc.dev/ontology/core#> .
```

#### semsrc:ContentAddressableObject
Base class for all content-addressed objects.

```turtle
semsrc:ContentAddressableObject a owl:Class ;
    rdfs:subClassOf prov:Entity ;
    rdfs:label "Content Addressable Object" ;
    rdfs:comment "An object addressed by its SHA-256 hash." ;
    semsrc:hasHash a owl:DatatypeProperty ;
        rdfs:domain semsrc:ContentAddressableObject ;
        rdfs:range xsd:string ;
        rdfs:comment "SHA-256 hash of object content" .
```

#### semsrc:StorageBackend
Representation of storage backend configuration.

```turtle
semsrc:StorageBackend a owl:Class ;
    rdfs:label "Storage Backend" ;
    rdfs:comment "Configuration for an object storage backend." ;
    semsrc:backendType a owl:DatatypeProperty ;
        rdfs:domain semsrc:StorageBackend ;
        rdfs:range [ a owl:DataOneOf ;
            owl:oneOf ( "filesystem" "s3" "azure" "memory" ) ] .
```

### Git-Compatible Classes

#### semsrc-git:Commit
Extends Git commit concept with semantic annotations.

```turtle
semsrc-git:Commit a owl:Class ;
    rdfs:subClassOf prov:Activity ;
    rdfs:label "Commit" ;
    rdfs:comment "A version control commit, extending Git commits with semantic metadata." ;
    semsrc-git:hasTree a owl:ObjectProperty ;
        rdfs:domain semsrc-git:Commit ;
        rdfs:range semsrc-git:Tree ;
        rdfs:comment "Commit points to root tree." ;
    semsrc-git:hasParent a owl:ObjectProperty ;
        rdfs:domain semsrc-git:Commit ;
        rdfs:range semsrc-git:Commit ;
        rdfs:comment "Parent commit(s) - allows multiple for merges." ;
    semsrc-git:commitMessage a owl:DatatypeProperty ;
        rdfs:domain semsrc-git:Commit ;
        rdfs:range xsd:string .
```

#### semsrc-git:Tree
Directory tree structure.

```turtle
semsrc-git:Tree a owl:Class ;
    rdfs:subClassOf fs:Directory, semsrc:ContentAddressableObject ;
    rdfs:label "Tree" ;
    rdfs:comment "Directory tree snapshot." .
```

#### semsrc-git:Blob
File content blob.

```turtle
semsrc-git:Blob a owl:Class ;
    rdfs:subClassOf fs:File, semsrc:ContentAddressableObject ;
    rdfs:label "Blob" ;
    rdfs:comment "File content blob." .
```

#### semsrc-git:Branch
Git branch representation.

```turtle
semsrc-git:Branch a owl:Class ;
    rdfs:label "Branch" ;
    rdfs:comment "A Git branch pointing to a commit." ;
    semsrc-git:pointsTo a owl:ObjectProperty ;
        rdfs:domain semsrc-git:Branch ;
        rdfs:range semsrc-git:Commit ;
        rdfs:comment "Branch points to specific commit." ;
    semsrc-git:branchName a owl:DatatypeProperty ;
        rdfs:domain semsrc-git:Branch ;
        rdfs:range xsd:string ;
        rdfs:comment "Name of the branch (e.g., 'main', 'develop')." .
```

#### semsrc-git:Tag
Git tag representation.

```turtle
semsrc-git:Tag a owl:Class ;
    rdfs:label "Tag" ;
    rdfs:comment "A Git tag pointing to a commit or tree." ;
    semsrc-git:pointsTo a owl:ObjectProperty ;
        rdfs:domain semsrc-git:Tag ;
        rdfs:range semsrc-git:Commit ;
        rdfs:comment "Tag points to specific commit." ;
    semsrc-git:tagName a owl:DatatypeProperty ;
        rdfs:domain semsrc-git:Tag ;
        rdfs:range xsd:string ;
        rdfs:comment "Name of the tag (e.g., 'v1.0.0')." ;
    semsrc-git:isAnnotated a owl:DatatypeProperty ;
        rdfs:domain semsrc-git:Tag ;
        rdfs:range xsd:boolean ;
        rdfs:comment "Whether tag is annotated with message and metadata." .
```

### Build System Classes

#### semsrc-build:Build
Represents a build execution.

```turtle
semsrc-build:Build a owl:Class ;
    rdfs:subClassOf prov:Activity ;
    rdfs:label "Build" ;
    rdfs:comment "A build execution." ;
    semsrc-build:buildsCommit a owl:ObjectProperty ;
        rdfs:domain semsrc-build:Build ;
        rdfs:range semsrc-git:Commit ;
        rdfs:comment "Build was for this commit." ;
    semsrc-build:producedArtifact a owl:ObjectProperty ;
        rdfs:domain semsrc-build:Build ;
        rdfs:range semsrc-build:Artifact ;
        rdfs:comment "Build produced this artifact." ;
    semsrc-build:status a owl:DatatypeProperty ;
        rdfs:domain semsrc-build:Build ;
        rdfs:range [ a owl:DataOneOf ;
            owl:oneOf ( "success" "failure" "running" "cancelled" ) ] .
```

#### semsrc-build:Artifact
Build output artifact.

```turtle
semsrc-build:Artifact a owl:Class ;
    rdfs:subClassOf prov:Entity ;
    rdfs:label "Build Artifact" ;
    rdfs:comment "Artifact produced by a build." ;
    semsrc-build:artifactType a owl:DatatypeProperty ;
        rdfs:domain semsrc-build:Artifact ;
        rdfs:range [ a owl:DataOneOf ;
            owl:oneOf ( "binary" "library" "package" "log" "coverage" ) ] .
```

### Ticket/Issue Classes

#### semsrc-ticket:Ticket
Issue/ticket tracking.

```turtle
semsrc-ticket:Ticket a owl:Class ;
    rdfs:subClassOf prov:Entity ;
    rdfs:label "Ticket" ;
    rdfs:comment "Issue or ticket tracked in the system." ;
    semsrc-ticket:ticketId a owl:DatatypeProperty ;
        rdfs:domain semsrc-ticket:Ticket ;
        rdfs:range xsd:string ;
    semsrc-ticket:status a owl:DatatypeProperty ;
        rdfs:domain semsrc-ticket:Ticket ;
        rdfs:range [ a owl:DataOneOf ;
            owl:oneOf ( "open" "in-progress" "review" "done" "closed" ) ] ;
    semsrc-ticket:fixedBy a owl:ObjectProperty ;
        rdfs:domain semsrc-ticket:Ticket ;
        rdfs:range semsrc-git:Commit ;
        rdfs:comment "Ticket was fixed by this commit." .
```

```turtle
### Tag Reference
:v1.0.0-tag a semsrc-git:Tag ;
    semsrc-git:tagName "v1.0.0" ;
    semsrc-git:pointsTo :commit-abc123 ;
    semsrc-git:isAnnotated true .
```

### Ticket-Commit Relationship
```turtle
:ticket-42 a semsrc-ticket:Ticket ;
    dcterms:title "Login page crashes on mobile" ;
    semsrc-ticket:ticketId "PROJ-42" ;
    semsrc-ticket:status "closed" ;
    semsrc-ticket:fixedBy :commit-abc123 .

:commit-abc123 prov:wasInformedBy :ticket-42 ;
    prov:qualifiedAssociation [
        a prov:Association ;
        prov:agent :alice ;
        prov:hadPlan :ticket-42
    ] .
```

### Build Provenance Chain
```turtle
:build-567 a semsrc-build:Build ;

    semsrc-build:status "success" ;
```

## OWL Constraints and Inferences

### Domain/Range Constraints
```turtle
# Only Commits can have parent commits
:semsrc-git:hasParent rdfs:domain semsrc-git:Commit .
:semsrc-git:hasParent rdfs:range semsrc-git:Commit .

# Builds must be for commits
:semsrc-build:buildsCommit rdfs:domain semsrc-build:Build .
:semsrc-build:buildsCommit rdfs:range semsrc-git:Commit .

# Tickets can only be fixed by commits
:semsrc-ticket:fixedBy rdfs:domain semsrc-ticket:Ticket .
:semsrc-ticket:fixedBy rdfs:range semsrc-git:Commit .
```

### Property Characteristics
```turtle
# Hashes are functional (each object has exactly one hash)
:semsrc:hasHash a owl:FunctionalProperty .

# Commit messages are functional
:semsrc-git:commitMessage a owl:FunctionalProperty .

# Build status is functional
:semsrc-build:status a owl:FunctionalProperty .
```

### Inferred Relationships
Using OWL reasoning, we can infer:

1. **Transitive provenance**: If commit A used ticket T, and build B built commit A, then build B indirectly used ticket T.

2. **Symmetric relationships**: If commit X fixed ticket Y, then ticket Y was fixed by commit X (inverse property).

3. **Class membership**: Any entity with a `semsrc-git:hasTree` property is inferred to be a `semsrc-git:Commit`.

## Validation Rules

### SHACL Shapes for Data Quality
```turtle
@prefix sh: <http://www.w3.org/ns/shacl#> .
@prefix semsrc-git: <http://semsrc.dev/ontology/git#> .

:CommitShape a sh:NodeShape ;
    sh:targetClass semsrc-git:Commit ;
    sh:property [
        sh:path dcterms:created ;
        sh:datatype xsd:dateTime ;
        sh:minCount 1 ;
        sh:maxCount 1 ;
        sh:message "Commit must have exactly one creation timestamp"
    ] ;
    sh:property [
        sh:path semsrc-git:commitMessage ;
        sh:minCount 1 ;
        sh:maxCount 1 ;
        sh:message "Commit must have a message"
    ] .
```

## Implementation Notes

### Serialization Formats
- **Development**: Turtle (.ttl) for readability
- **Storage**: N-Triples for simplicity and streaming
- **Exchange**: JSON-LD for web APIs

### Namespace Management
- Use consistent, versioned namespace URLs
- Provide namespace declaration files
- Support prefix.cc registration for well-known prefixes

### Versioning Strategy
- Ontology versions follow semantic versioning
- Breaking changes require new namespace
- Deprecated terms marked with `owl:deprecated true`
- Migration tools for existing data

### AI Considerations
- Use `rdfs:label` and `rdfs:comment` extensively
- Link to external vocabulary definitions
- Provide example SPARQL queries
- Document common inference patterns
